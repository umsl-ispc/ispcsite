Processing an ASP File
• When ASP receives a page from IIS that contains server-side script
code, it parses it line-by-line. Anything that is not server-side script,
is sent back to IIS, and onwards to the client. As each section of
script is reached, it is passed to the appropriate scripting engine.
The results of which are inserted at the appropriate points as it is
sent to IIS.
• Script is identified by either the <% %> script delimiters or by
using the <SCRIPT> element.
• Pages have an extension of .asp
• The default scripting language is VBScript, however Jscript is a
suitable alternative.
– VBScript is NOT Visual Basic but is, in fact, a subset of the actual
language.
– Jscript IS based on Netscape’s JavaScript with the added feature of
being able to be run at the server.
