The Basics of Server Side Scripting
• For a script to work with a Web Server there needs to be
some kind of intermediate application, or add-in, to
connect the two. It has to accept a request from the
user, read and interpret the appropriate server-based
script file, and then create the output page and
communicate it to the Web server where it is sent as the
response to the client.
• Typically the task is divided into two sections.
– One application or add-in handles the communication to and
from the Web server. (CGI)
– Another handles interpreting and executing the script.
• Perl was the first such popular server-side scripting
language.
